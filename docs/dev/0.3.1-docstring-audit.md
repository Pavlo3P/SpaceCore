# SpaceCore 0.3.1 public docstring audit

Audit command:

```bash
.venv312/bin/python scripts/docstring_audit.py
```

Result: `numpydoc issues: 0` for top-level public exports after excluding the
allowed migration-baseline codes configured in the script.

## Example coverage baseline

A top-level export scan found 82 public classes/functions with docstrings and 10
with explicit `Examples` sections.

Objects with examples:

- `Context`
- `LinOp`
- `DiagonalLinOp`
- `DenseLinOp`
- `SparseLinOp`
- `cg`
- `lsqr`
- `lanczos_smallest`
- `power_iteration`
- `expm_multiply`

Objects without explicit examples are marked `missing example` for the 0.3.1
baseline. They remain prose-covered and numpydoc-clean, but should receive tiny
examples in follow-up docstring-only patches.

## Module checklist

## Exceptions and follow-ups

No public top-level object is impossible to demonstrate in 10 lines or fewer.
The missing examples are follow-up work rather than infeasible cases.
